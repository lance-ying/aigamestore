- p.logs is write-only. Track all logs since start for both HUMAN and TEST modes:
	- "game_info": gamePhase/controlMode changes
		- "game_status": gameState.gamePhase
		- "data": Additional state data (e.g., score when game over)

	- "inputs": Control inputs when triggered
		- "input_type": Event type (keyPressed, keyReleased, etc.)

	- "player_info": Player state changes (gameState.player)
		- "screen_x", "screen_y": Player screen position
		- "game_x", "game_y": Player world position
