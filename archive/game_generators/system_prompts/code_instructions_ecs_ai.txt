<ecs_code_architecture>
Use Entity-Component-System (ECS) architecture with ES6 modules:
- **Single Source of Truth**: Store all entities in ONE global array
- **Components**: Pure data structures
- **Systems**: Logic that operates on entities with specific components
- **Proper Imports**: Every file MUST import ALL variables, functions, and classes it uses
  <example_code>
  // Essential pattern (ALWAYS follow this)
  import { PositionComponent, VelocityComponent } from './components.js';
  import { entities, gameState } from './globals.js';
  import { get_ai_action } from './ai_controller.js';

  // Verify EVERY function has access to ALL components it needs
  function createEntity(x, y) {
    const entity = {
      components: {
position: new PositionComponent(x, y) // Must import PositionComponent!
      }
    };
    entities.push(entity);
    return entity;
  }
  </example_code>
- **State Management**: 
  - Use a single `gameState` object to track the game's current data and status. Store all game data in the `gameState` object which would be required for an automated AI testing to play test the game.
  <example_code>
  export const entities = []; // canonical array including player
  export const gameState = {
    player: null,
    score: 0,
    entities, // alias to canonical array
    gamePhase: "START", // "START", "PLAYING", "GAME_OVER_WIN/LOSE", "PAUSED"
    controlMode: "HUMAN", // "HUMAN", "AI_WIN", "AI_TEST_A", "AI_TEST_B", etc.
  };
  </example_code>
  - Implement `function getGameState()`; attach it to `window`. It **must** return the `gameState` object.
</ecs_code_architecture>

<game_mechanics_instructions>
Game mechanics define the rules of the game: controls, behavior of entities, interactions between entities and the environment, physics, rewards, and win/lose conditions.
Ensure that the game mechanics are implemented to be intuitive, believable, and fun enabling the player to achieve the final goal. Your implementation will be tested by the automated AI testing modes so ensure that the game mechanics are implemented correctly.
- **Controls**:
  - **Controls & Key Mapping**: Only keyboard keys as inputs. NO MOUSE based controls. Use these keycodes: Arrow keys (37, 38, 39, 40), SPACE (32), Z (90), SHIFT (16), ENTER (13): Start game (from START screen), R (82): Restart game (from GAME_OVER screens), ESC (27): Pause/Unpause game
  - **Game Phases Transitions Keys**: START → PLAYING: ENTER, PLAYING → GAME_OVER: win/lose, GAME_OVER → START: R, PLAYING ↔ PAUSED: ESC
  - **Action Mapping**: Define the action mapping for the game mechanics and their effects on the player state
  - **Input Handling**: 
    - Implement input handling to map the keyboard keys to the game mechanics.
    - If controlMode is HUMAN, then the input handling should be based on the keyboard keys.
    - When performing AI testing, the input handling should be based on action outputs from selected AI mode's function
    <example_code>
    export function inputSystem(gameState) {
        // Get input action based on control mode
        let action;
        if (gameState.controlMode === "HUMAN") {
            // Map keyboard input to game actions
            action = getHumanInput(gameState);
        } else {
            // Get AI action from the current AI mode's controller
            action = get_ai_action(gameState);
        }
        return action;
    }
    </example_code>

- **Physical Interactions**:

- **Win/Lose Conditions**:

  - Show the game over screen when the game is over
- **Rewards & Punishments**:
  - Define the rewards and effects of interactions with the game entities and the environment
