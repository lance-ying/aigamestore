<instructions>
<hard_constraints>
- Code Implementation in JavaScript using p5.js following the Entity-Component-System (ECS) architecture.
- Game Mechanics & Controls must be implemented using keyboard keys only. NO MOUSE based controls. Game mechanics must allow the player to achieve the final goal with interesting subgoals to maintain progress.
- Physics must be consistent to carry out the game mechanics. Do not have objects pass through each other or any other inconsistencies.
- Graphics & Animations must be smooth and polished, implemented using p5.js primitives. NO external images, sprites, or fonts. No audio or sound effects. 
- Automated AI testing: Provide one “AI_WIN” and maximum 4 additional test modes that call `get_ai_action()` each frame to test different aspects of the game.
- Reproducibility: Ensure game reproducibility using p.randomSeed(42) in the setup function.
- Game Initialization: When initializing the game, ensure that the positions of the entities and other game elements allow for the player to settle in the game and not be immediately killed, blocked, etc.
- Environment layout: Ensure that the environment layout, entities and game elements are such that the player can make progress and achieve the game objectives.
</hard_constraints>

<concept_to_design>
A game is an interactive experience between the player and the environment, governed by mechanics, constraints, rewards, objectives, and a final goal. A good game has a well-paced learning curve, progression, and rewards to guide the player to achieve the final goal.

Process: 
Begin by parsing the <game_concept>. Identify what is defined: narrative setting, entities, mechanics, goals, viewpoint, etc. Then, list missing or underspecified elements -- this is where your creativity and design expertise is essential.
Curate the game design by adding elements that make it fun and playable:
- New mechanics or combinations of interactions
- Subgoals and optional challenges
- Win/loss conditions or score thresholds
- Adversaries, moving obstacles, collectibles

<example>
If the game concept is: “Make me a game where a spaceship explores space,” you might define:
- The player collects gemstones from 10 planets while evading enemy ships.
- The ship must shoot aliens in a side-scrolling environment to survive.
- The objective is to navigate an asteroid field to reach a distant planet.
</example>

Express the game design with your additions to the game concept in a structured design process to design the game meaningfully before implementation:
- Define the narrative and overall objective.
- Describe the game world and spatial layout.
- List all entities: player character, NPCs, enemies, obstacles, collectibles, and how they interact.
- Map keyboard input to mechanics. Define interactions, collisions, and physics.
- Structure player progression using subgoals, checkpoints, and escalating difficulty.
- Define the final goal: what constitutes success? What ends the game?
- Design visual elements consistent with the game’s theme and mechanics. Use smooth animations, artistic graphics, and vibrant color schemes.

Definitely include all the elements required by the <game_concept> but feel free to add more elements to make a compelling game. The gameplay experience should be interesting with achievable goals.
</concept_to_design>

<code_instructions>
You are encouraged to write as much code as you can to make the game more interesting and aesthetically pleasing.

<code_architecture>
Use Entity-Component-System (ECS) architecture with ES6 modules:
- Single Source of Truth: Store all entities in ONE global array
- Components: Pure data structures
- Systems: Logic that operates on entities with specific components
- Organize the code into multiple files as needed (game.js, globals.js, etc.)
- Proper Imports: 
  - Use ES6 for imports and exports.
  - Every file MUST import ALL constants, variables, functions, classes, and external symbols at the top of every file. No Node.js style require imports.
<example_code>
// Essential pattern (ALWAYS follow this)
import { PositionComponent, VelocityComponent } from './components.js';
import { entities, gameState, CANVAS_WIDTH } from './globals.js';

// Verify EVERY function has access to ALL components it needs
function createEntity(x, y) {
  const entity = {
    components: {
position: new PositionComponent(x, y) // Must import PositionComponent!
    }
  };
  entities.push(entity);
  return entity;
}
</example_code>

- State Management: 
  - Use a single `gameState` object to track the game's current data and status. Store all game data in the `gameState` object which will be used by automated AI testing code to play-test the game.    
  - Implement `function getGameState()`; attach it to `window`. It must return the `gameState` object.

<example_code>
export const entities = []; // canonical array including player
export const gameState = {
  player: null,       // player entity to be initialized in the setup function and initialized before the game starts
  score: 0,
  entities,           // alias to canonical array
  gamePhase: "START", // "START", "PLAYING", "GAME_OVER_WIN/LOSE", "PAUSED"
  controlMode: "HUMAN", // "HUMAN", "AI_WIN", "AI_TEST_A", "AI_TEST_B", etc.
};
</example_code>

<p5_instructions>
- Use p5.js in instance mode and store the p5 instance in a variable called `gameInstance`. Expose the game instance globally as follows:
  <example_code>
  const p5 = window.p5
  let gameInstance = new p5(p => {
    p.randomSeed(42); // ensures reproducibility.
    ...
  });
  // Expose the game instance globally
  window.gameInstance = gameInstance;
  </example_code>
- Useful Functions:
  - Use p. to call p5 functions.
  - p5: `keyIsDown(key_code)`, `noLoop()/loop()`, `tint()`, `push()/pop()`, `textAlign()`, `textSize()`, `text()`
  - p5.collide2d for collision detection
    - Note that the specific order of the words in the function name matters. For example, 'collideCircleRect' is not available.
    - Available functions: `collidePointPoint`, `collidePointCircle`, `collidePointEllipse`, `collidePointRect`, `collidePointLine`, `collidePointArc`, `collideRectRect`, `collideCircleCircle`, `collideRectCircle`, `collideLineLine`, `collideLineCircle`, `collideLineRect`, `collidePointPoly`, `collideCirclePoly`, `collideRectPoly`, `collideLinePoly`, `collidePolyPoly`, `collidePointTriangle`.
</p5_instructions>

<game_mechanics_instructions>
Game mechanics define the rules, controls, interactions, physics, rewards, and win/lose conditions. Implement intuitive, believable mechanics that enable players to achieve the final goal.

- Controls:
  - Allowed keys: Arrow keys (37-40), SPACE (32), Z (90), SHIFT (16), ENTER (13), R (82), ESC (27)
  - Phase transitions: START→PLAYING (ENTER), PLAYING→GAME_OVER (win/lose), GAME_OVER→START (R), PLAYING↔PAUSED (ESC)
  - Input handling: Use keyboard for HUMAN mode, AI action outputs for AI testing modes

- NPC behavior:
  - Implement intelligent, challenging behavior for NPCs and game entities
  - Ensure proper entity interactions

- Physical Interactions:
  - Base interactions on plausible physics with consistent parameters
  - Prevent objects from passing through each other or behaving inconsistently

- GAME_OVER Conditions:
  - Implement achievable win conditions and appropriate lose conditions

- Rewards:
  - Define meaningful rewards for interactions
  - Create subgoals for progression toward the final objective
</game_mechanics_instructions>

<rendering_instructions>
- Canvas Setup: 600×400px canvas, 60 FPS
- Ensure the correct render order of the game objects
- Implement smooth aesthetics, animations, and effects consistent with the game environment, entities, and their interactions
- Add well positioned text or graphics to inform the player about information like score, health, etc.
- Avoid flickering:
  - Do not call p.randomSeed() inside draw()
  - Have exactly one call to p.background() at the top of draw()
  - Do not draw elements that are randomly sampled at every frame as this causes flickering
- Use ternary operator to set colors for conditional rendering: Example: `p.fill(...(CONDITION ? [255, 220, 150] : [40, 30, 20]));`
- Never display current controlMode (HUMAN/AI) on the game canvas.
- Required Game Screens:
  - Start: Title, instructions, "PRESS ENTER TO START"
  - Playing: Game environment, score, and active gameplay
  - Pause: Small text "PAUSED" indicator in top right
  - Game Over: Win/lose message, final score, "PRESS R TO RESTART"
</rendering_instructions>
