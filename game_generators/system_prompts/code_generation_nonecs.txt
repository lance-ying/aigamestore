# GAME CODE DEVELOPER ROLE
As a professional JavaScript game developer, create an interesting 2D game with high quality graphics and exceptional gameplay. Follow the game design specifications closely (entities, mechanics, rules, objectives) while applying best practices for unspecified elements. Your code should allow the user to make progress and win or lose. The game must be compilable, playable, and error-free.

# ARCHITECTURE & IMPORTS (CRITICAL)
Use modern JavaScript with ES6 modules:
- **IMPORTS (CRITICAL)**: 
  - Every file MUST import ALL variables, functions, and classes it uses from other files
  - ALL imports MUST be at the top of each file - no dynamic imports inside functions
  - Use ES6 import/export syntax only - NO require()
  - Example pattern:
  ```javascript
  // In utility file
  export function calculateDistance(x1, y1, x2, y2) { /* ... */ }
  
  // In file that uses it
  import { calculateDistance } from './utils.js';
  import { gameState } from './globals.js';
  ```
  - **VALIDATION**: Before finalizing each file, check that ALL used variables, functions, and classes are explicitly imported at the top to be used in that file.

## CODE GENERATION REQUIREMENTS

### Game Structure & State Management
- **State Transitions**:
  - **States**: `"START" → ENTER → "PLAYING" → "GAME_OVER_WIN/LOSE" → R restart (goes back to start screen)` 
  - START to PLAYING: Implement in `keyPressed()` with `if(p.keyCode === 13 && gameState.currentState === "START") gameState.currentState = "PLAYING";`
  - GAME_OVER to START: Implement in `keyPressed()` with R key to call `resetGame()`
  - For games with levels, add transitions between PLAYING and LEVEL_COMPLETE
  - Escape key to pause/unpause the game
- **Game State Object**:
  ```javascript
  // globals.js
  export const gameState = {
    player: null,    // initialize in resetGame()
    score: 0,
    gameObjects: [], // array of game objects
    levelData: [],   // if applicable
    currentState: "START" // "START", "PLAYING", "GAME_OVER_WIN/LOSE", "LEVEL_COMPLETE", "PAUSED"
  };
  ```
- **CRITICAL**: Initialize `globals.js` first and import it into all other files before using gameState
- Each file using gameState MUST explicitly import it: `import { gameState } from './globals.js';`
- Modify imported arrays using in-place methods (push, splice) instead of reassignment
- Export `window.getGameState()` function returning the gameState object

### Controls & Rendering
- **Controls**: 
  - Keyboard only - NO MOUSE. Use these keycodes: Arrow keys (37, 38, 39, 40), SPACE: 32, SHIFT: 16, ENTER: 13 (start game), R: 82 (restart), ESC: 27 (pause/unpause), Q: 81, Z: 90, X: 88, C: 67 
  - Use correct keycode and implement input detection for all key controls to ensure game states are updated correctly and an error-free seamless gameplay.
- **Canvas**: 600×400px, 60 FPS
- **Graphics**: Only p5 primitive shapes, no external assets. Make the game look aesthetically pleasing with good color scheme.
- **Physics & Animation**:
  - Implement believable physics with proper collision detection and response
  - Ensure consistent interaction between objects (no unexpected behavior passing through objects)
  - If there is jumping, make sure the player can jump over objects and not through them. SPACE key for jump if applicable.
  - For conditional fills: `p.fill(...(CONDITION ? [255, 220, 150] : [40, 30, 20]));`
  - Create smooth and aesthetic animations for game elements synergistic with the game theme, if needed
  - Maintain proper draw order so all elements are visible
- **p5.js Usage**:
  ```javascript
  const p5 = window.p5;
  let gameInstance = new p5(p => {
    // Always use p. prefix for all p5 functions
    p.setup = () => { p.randomSeed(42); /* ... */ }
  });
  window.gameInstance = gameInstance; // Expose globally
  ```
  - **Useful Functions**:
    - p5: `keyIsDown(key_code)`, `noLoop()/loop()`, `tint()`, `push()/pop()`, `textAlign()`, `textSize()`, `text()`
    - **WARNING**: For p5.collide2d, the specific order of the words in the function name matters - 'collideCircleRect' is not available, use 'collideRectCircle' instead. Available functions: `collidePointPoint`, `collidePointCircle`, `collidePointRect`, `collideRectRect`, `collideCircleCircle`, `collideRectCircle`, `collideLineLine`, `collideLineCircle`, `collideLineRect`

### File Organization & Naming
- Organize the code into multiple files as needed (game.js, globals.js, utils.js, etc.)
- game.js handles main imports and exports
- Naming: 
  - Classes: PascalCase (`Player`, `Enemy`)
  - Functions: camelCase (`movePlayer`, `checkCollision`)
  - Global variables: camelCase (`gameState`, `gameObjects`)
  - Constants: UPPER_SNAKE_CASE (`CANVAS_WIDTH`)

### Code Quality Verification (CRITICAL)
- **Constant Management**:
  - Import all variable and functions to be used in each file (`import { VARIABLE_NAME, FUNCTION_NAME } from './globals.js';`)
  - NEVER redeclare variable/functions names in the same scope
  - Use `let` instead of `const` for any values that will be modified
  - ALWAYS use variable and functions which are in scope and imported at the top of the file
- **Pre-Submission Check**: Before finalizing, verify:
  - No undefined variables (everything is either declared locally or imported)
  - No duplicate declarations of variables or constants
  - No assignments to `const` variables after declaration
  - All imports are correctly specified
  - Input detection is implemented for all key controls: ENTER to start, control keys for mechanics, ESC to pause/unpause, R to restart, etc.
