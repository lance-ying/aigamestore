# GAME CODE DEVELOPMENT GUIDELINES
Implement the game using Entity-Component-System (ECS) architecture. Store all entities in one global array for external inspection. Components should be data-only structures with no logic. Systems implement game logic and operate on entities with their components. If scoring is required, include a `ScoreComponent` in the `player` entity; otherwise, initialize it with a value of 0. Maintain a SINGLE source of truth for the entities array across all modules.

## KEY TECH SPECS
- Game states: `"START" → ENTER (13) → "PLAYING" → "GAME_OVER_WIN" or "GAME_OVER_LOSE" → R (82) restart`. For games with levels, include a `"LEVEL_COMPLETE"` state requiring ENTER to advance. Implement resetGame() and resetLevel() functions to handle transitions.
- Controls: ONLY keyboard inputs (NO MOUSE). Allowed keys with ASCII codes: Arrow keys (37-40), SPACE (32), SHIFT (16), Q (81), Z (90), X (88), C (67), ENTER (13), R (82), ESCAPE (27). Only use Enter to start, R to restart, Escape to pause/unpause.
- Mechanics: Implement mechanics that provide a smooth and enjoyable gameplay experience with correct key input handling, believable physics in game mechanics, and plausible behaviors for game elements catering to the game design.
- Rendering:
  - Canvas: `600 × 400 px`, target 60 FPS.
  - NO external images, audio, or fonts. Use primitive p5 shapes to represent entities and objects in the environment.
  - For conditional fills: `p.fill(...(CONDITION ? [255, 220, 150] : [40, 30, 20]));`
  - Create animated elements as needed for aesthetics and game appearance without breaking the game mechanics. The animations should be smooth and synergistic with the game theme and mechanics.
  - Avoid randomly sampling elements every frame to prevent flickering.
  - Ensure all UI elements and entities are visible within the canvas with proper formatting and positioning.
  - Maintain appropriate draw order so the player is always visible and the game is rendered correctly.
- JavaScript module and variables:
  - Use p5.js in instance mode with variable `gameInstance` and expose it globally:
    ```javascript
    const p5 = window.p5;
    let gameInstance = new p5(p => {
        ...
    });
    // Expose the game instance globally
    window.gameInstance = gameInstance;
    ```
  - Always pass the object `p` to access p5js functions and avoid "ReferenceError: p is not defined".
  - Set random seed for reproducibility (`p.randomSeed(42)` in setup).
  - Write helper functions to avoid duplication, handle errors, and edge cases.
  - Split code into multiple files as needed (game.js, player.js, enemy.js, world.js, etc.) with game.js handling all imports.
  - Export all necessary objects such as gameState, etc. from game.js if you need to expose them to the global scope: `export { gameState }` 
  - Keep ALL game state in one exported object:
    ```javascript
    // globals.js
    export const entities = []; // canonical array including player
    export const gameState = {
      player: null,    // must be initialized in resetGame() in game.js
      score: 0,        // accessible via player.ScoreComponent
      entities,        // alias containing player entity
      levelData: [],   // only if there are levels
      currentState: "START" // "START", "PLAYING", "PAUSED", "LEVEL_COMPLETE", "GAME_OVER_WIN", or "GAME_OVER_LOSE"
    };
    // In other files:
    import { gameState, entities, MAX_SPEED } from './globals.js';
    // Access as: gameState.entities, gameState.player
    ```
  - Use ES6 modules (import/export) instead of Node.js require().
  - Define shared variables in a single source file. Import all necessary variables, classes, and functions at the top of each file.
  - Modify imported arrays using in-place methods (push, splice, pop) instead of reassignment.
  - Implement `function getGameState()` and attach to `window`. It must return the gameState object.
  
## NAMING GUIDELINES
- ENTITY: PascalCase noun + some_number for multiples (`Player`, `Enemy`, `Platform0`)
- COMPONENT: noun + `Component` (`PositionComponent`, `HealthComponent`)
- SYSTEM: verb/noun + `System` (`MovementSystem`, `CollisionSystem`)
- GLOBAL LISTS: plural camelCase (`entities`)
- CONSTANTS: `UPPER_SNAKE_CASE` (`CANVAS_WIDTH`)

## USEFUL FUNCTIONS
Use functions from the p5.js and p5.collide2D libraries and avoid reinventing the wheel. These functions are accessible through the `p` object.
- p5 functions: `keyIsDown(key_code)`, `keyPressed(key_code)`, `noLoop()/loop()`, `tint()`, `push()/pop()`, `textAlign(CENTER,CENTER)`, `textSize(12)`, `text(text, x, y)`, etc.
- Use p5.collide2D for ALL collision detection. Available functions: `collidePointPoint`, `collidePointCircle`, `collidePointEllipse`, `collidePointRect`, `collidePointLine`, `collidePointArc`, `collideRectRect`, `collideCircleCircle`, `collideRectCircle`, `collideLineLine`, `collideLineCircle`, `collideLineRect`, `collidePointPoly`, `collideCirclePoly`, `collideRectPoly`, `collideLinePoly`, `collidePolyPoly`, `collidePointTriangle`. Note that the specific order of the words in the function name matters. For example, 'collideCircleRect' is not available.
