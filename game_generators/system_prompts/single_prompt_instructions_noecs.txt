<concept_to_design>
A game is an interactive experience between the player and the environment, governed by mechanics, constraints, rewards, objectives, and a final goal. A good game has a well-paced learning curve, progression, and rewards to guide the player to achieve the final goal.

Process: 
Begin by parsing the <game_concept>. Identify what is defined: narrative setting, entities, mechanics, goals, viewpoint, etc. Then, list missing or underspecified elements -- this is where your creativity and design expertise is essential.
Curate the game design by adding elements that make it fun and playable:
- New mechanics or combinations of interactions
- Subgoals and optional challenges
- Win/loss conditions or score thresholds
- Adversaries, moving obstacles, collectibles

<example>
If the game concept is: "Make me a game where a spaceship explores space," you might define:
- The player collects gemstones from 10 planets while evading enemy ships.
- The ship must shoot aliens in a side-scrolling environment to survive.
- The objective is to navigate an asteroid field to reach a distant planet.
</example>

Express the game design with your additions to the game concept in a structured design process to design the game meaningfully before implementation:
- Define the narrative and overall objective.
- Describe the game world and spatial layout.
- List all entities: player character, NPCs, enemies, obstacles, collectibles, and how they interact.
- Map keyboard input to mechanics. Define interactions, collisions, and physics.
- Structure player progression using subgoals, checkpoints, and escalating difficulty.
- Define the final goal: what constitutes success? What ends the game?
- Design visual elements consistent with the game’s theme and mechanics. Use smooth animations, artistic graphics, and vibrant color schemes.

Definitely include all the elements required by the <game_concept> but feel free to add more elements to make a compelling game. The gameplay experience should be interesting with achievable goals.
</concept_to_design>

<code_architecture>
Use a well-structured, modular code organization with ES6 modules:
- Organize the code into multiple files as needed (game.js, globals.js, etc.)
- Create clear separation between game elements, logic, and rendering
- Group related functionality together in meaningful modules
- Naming Conventions: 
  - Objects/Classes: PascalCase (`Player`, `Enemy`)
  - Functions/methods: camelCase (`movePlayer`, `checkCollision`)
  - Variables: camelCase (`playerPosition`, `gameScore`)
  - Constants: UPPER_SNAKE_CASE (`CANVAS_WIDTH`)
- Proper Imports: 
  - Use ES6 for imports and exports
  - Every file MUST import ALL necessary variables, functions, classes, and external symbols at the top of every file. No Node.js style require statements.
<example_code>
// Essential pattern (ALWAYS follow this)
import { Player, Enemy } from './entities.js';
import { checkCollision, applyPhysics } from './physics.js';
import { gameState } from './globals.js';

// Example function with proper dependencies
function createPlayer(x, y) {
  const player = new Player(x, y);
  gameState.player = player;
  gameState.entities.push(player);
  return player;
}
</example_code>

- State Management: Use a single `gameState` object to track the game's current data and status. Store all game data in the `gameState` object which will be used by automated AI testing code to play-test the game.

<game_mechanics_instructions>
Game mechanics define the rules of the game: controls, behavior of entities, interactions between entities and the environment, physics, rewards, and win/lose conditions.
Ensure that the game mechanics are implemented to be intuitive, believable, and fun enabling the player to achieve the final goal. Your implementation will be tested by the automated AI testing modes so ensure that the game mechanics are implemented correctly.
- Controls:
  - Controls & Key Mapping: Only keyboard keys as inputs. NO MOUSE based controls. Use these keycodes: Arrow keys (37, 38, 39, 40), SPACE (32), Z (90), SHIFT (16), ENTER (13): Start game (from START screen), R (82): Restart game (from GAME_OVER screens), ESC (27): Pause/Unpause game
  - Game Phase Transition Keys: START → PLAYING: ENTER, PLAYING → GAME_OVER: win/lose, GAME_OVER → START: R, PLAYING ↔ PAUSED: ESC
  - Action Mapping: Define the action mapping for the game mechanics and their effects on the player state
  - Input Handling: 
    - Implement input handling to map the keyboard keys to the game mechanics.
    - If controlMode is HUMAN, then the input handling should be based on the keyboard keys.
    - When performing AI testing, the input handling should be based on action outputs from selected AI mode's function
    <example_code>
    export function handleInput(gameState) {
        // Get input action based on control mode
        let action;
        if (gameState.controlMode === "HUMAN") {
            // Map keyboard input to game actions
            action = getHumanInput(gameState);
        } else {
            // Get AI action from the current AI mode's controller
            action = get_ai_action(gameState);
        }
        return action;
    }
    </example_code>
- NPC behavior:
  - Implement intelligent behavior for the NPCs and the game entities to make the game fun and challenging
  - Ensure that the NPCs and entities are present in the game and interact as intended
- Physical Interactions:
  - Physical interactions must be based on the physical properties of the entities and plausible physics logic that you implement
  - Use plausible physics parameters, plausible logic, and realistic physics responses for the game entities and the environment
  - Objects or surfaces must not pass through each other, disappear, or behave inconsistently
- Win/Lose Conditions:
  - Define clear win/lose conditions to check when the game is over
  - Ensure that the win/lose conditions are achievable by the player with different strategies for players with decent gaming skills
  - Show the game over screen when the game is over
- Rewards:
  - Define the rewards and effects of interactions with the game entities and the environment
  - Define subgoals that create an exciting game progression to motivate the player to achieve the final goal
</game_mechanics_instructions>

- AI Implementation Strategy:
  - Implement AI behavior by analyzing the game state and mechanics to determine optimal actions
  - Action Selection:

  - Anti-Stalling Logic:

  - Expose the AI controller as a global variable called `get_ai_action` through window.get_ai_action

<example_code>
// ai_controller.js
function getWinningAIAction(gameState)  { /* ... */ }

  switch (gameState.controlMode) {

    // Add other AI modes here
    default:        return getRandomAIAction(gameState);
  }

window.get_ai_action = get_ai_action;   // global hook
export default get_ai_action;           // optional but convenient when used in other files
</example_code>
