<instructions>
<hard_constraints>
- The game must load without errors, be responsive to player inputs, and have a fun gameplay loop.
- Design a progressive gameplay loop with a well-paced learning curve to assist in learning the game mechanics and controls to feeling challenged and eventually win the game.
- Map keyboard keys to player actions correctly that result in changes in game state. NO mouse-based controls. Allowed keys: Arrow keys (37-40), SPACE (32), Z (90), SHIFT (16), ENTER to start the game (13), R to restart the game after a win/loss (82), ESC to pause the game (27). 
- Output error-free code with proper imports, function logic, and variable declarations following best game development practices for p5.js.
- Rendering must have well-designed, vibrant graphics and dynamic informative animations for the environment, entities, and their interactions.
- As you write the game code, keep in mind that automated game testing functions will be implemented to validate different game mechanics and features of the game. Winning mode is required where the implemented function plays the game to win.
- Game should be reproducible. Set the random seed using p.randomSeed(42) in the setup function.
</hard_constraints>

<process_game_concept_to_game_design>
A game is an interactive experience between the player and the environment, governed by game mechanics that follow a progression.
The journey of a player through their interaction with the game starts with exploration and ends with achieving the final goal exploiting what they have learned.
Game progression is how difficulty based on mechanics, systems, narrative and rewards work together to create a fun experience.
A good game has a well-curated progression, well-paced learning curve, leading to learning about the game elements and mechanics which eventually allows the player to achieve the final goal.
Then, the game must be such that the player can win by showing their learned skill and not impossible to win.

Curate the game design with your additions beyond the game concept that is feasible to implement in code using p5.js following the hard constraints. Following are some game elements that you can think of:
- Narrative: the story of the game
- Environment: setting, viewpoint
- Entities: characters: player, NPCs, objects: collectibles, interactive objects, power-ups, destructibles, obstacles etc.
- Mechanics: character abilities, object interactions, rewards, subgoals, final goal
- Graphics:
  - Visual appearance: the visual appearance of the game environment and entities
  - Animations: the animations of the game entities and their interactions
- Progression:
  - Start with a trivial version of gameplay that acts as a tutorial to allow players to learn mechanics and understand the game with achievable challenges. Player must get 
  - Once the player has comfortably achieved those challenges, you can start introducing new game elements and mechanics to increase complexity at a reasonable pace
  - Maintain a sense of progression that feels natural and achievable throughout the entire gameplay experience
  - You must provide the player with a clear sense of progression and increasing challenges to allow them to demonstrate their skills to eventually win the game
  - Avoid abrupt difficulty spikes that could lead to surprise, frustration, or unexpected behavior
Include all the elements required by the game concept but feel free to add more elements to make a compelling game. You can add reasonable strategies that the player might use to make progress and win the game.
</process_game_concept_to_game_design>

<code_instructions>

<code_architecture>
- Modular code organization with proper ES6 imports:
  - Organize the code into multiple files as needed (game.js, globals.js, etc.)
  - Create clear separation between game elements, logic, and rendering
  - Group related functionality together in meaningful modules
  - Proper Imports:
    - Use ES6 for imports and exports
    - Every file MUST import ALL constants, variables, functions, classes, and external symbols at the top of every file. No Node.js style require imports. No dynamic imports within functions

<game_mechanics_instructions>
Implement intuitive, believable mechanics that enable players to achieve the final goal.
Choose parameters and logic for game mechanics that are intuitive and believable. These can change as the game progresses. 
Even at the end of the game, the mechanics must be such that the player can win by showing their learned skill.
- Controls:
  - Allowed keys: Arrow keys (37-40), SPACE (32), Z (90), SHIFT (16), ENTER (13), R (82), ESC (27)
  - Phase transitions: START→PLAYING (ENTER), PLAYING→GAME_OVER (win/lose), GAME_OVER→START (R), PLAYING↔PAUSED (ESC)
  - Input handling: Use keyboard for HUMAN mode, automated testing action outputs for automated testing modes
  - Player controls must be intuitive and mapped correctly to appropriate changes in game state
  - When automated testing mode is selected, the game must automatically play using the action outputs from the automated testing controller
- Non-player characters and objects:
  - Implement intelligent, intuitive, and challenging behavior for all game entities to keep the player engaged
  - Ensure proper entity interactions and responsive behavior consistent with their roles and objectives to give player feedback responsive to their actions
  - Entity properties and behavior must be such that the player can make progress in the game
  - Do not add excessive number of entities to prevent screen clutter and ensure understandable and manageable gameplay
- Physical Interactions:
  - Base interactions on plausible physics with parameters and logic to it a believable realistic feel
  - Prevent objects from passing through each other or behaving inconsistently to prevent shocking the player
- GAME_OVER Conditions:
  - Implement mechanics that can lead to win or lose conditions achievable by the player
  - GAME_OVER must be a final state followed by game over screen
- Rewards:
  - Define meaningful and reasonable conditions for rewards to keep the player motivated to reach the final goal

<rendering_instructions>
- Canvas Setup: 600×400px canvas, 60 FPS
- Ensure the correct render order of the game objects to ensure player is visible on the screen
- Implement aesthetic objects, smooth animations, and dynamic effects for the game environment, entities, and their interactions
- Provide useful visual feedback:
  - Player character must be distinctly identifiable
  - Provide visual feedback for interactions and response to their actions
  - Add well-positioned text and graphics to inform the player about information like score, health, etc.
  - Never display current controlMode (HUMAN/TESTING) on the game canvas.
- Avoid distractions and overburdening the player with too many elements on the screen
- Avoid flickering:
  - Do not call p.randomSeed() inside draw()
  - Have exactly one call to p.background() at the top of draw()
  - Do not draw elements that are randomly sampled at every frame as this causes flickering
- Use ternary operator to set colors for conditional rendering: Example: `p.fill(...(CONDITION ? [255, 220, 150] : [40, 30, 20]));`
- Required Informative Game Screens:
  - Start: Title, clear instructions, objectives, and controls with actions for the player, "PRESS ENTER TO START"
  - Playing: Game rendering, score, and active gameplay
  - Pause: Small text "PAUSED" indicator in top right
  - Game Over: Win/Lose message, final score, "PRESS R TO RESTART", R takes you back to the start screen
</rendering_instructions>
