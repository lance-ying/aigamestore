<process_game_concept_to_game_design>
A game is an interactive experience between the player and the environment, governed by game mechanics that follow a progression.
The journey of a player through their interaction with the game starts with exploration and ends with achieving the final goal exploiting what they have learned.
Game progression is how difficulty based on mechanics, systems, narrative and rewards work together to create a fun experience.
A good game has a well-curated progression, well-paced learning curve, leading to learning about the game elements and mechanics which eventually allows the player to achieve the final goal.
Then, the game must be such that the player can win by showing their learned skill and not impossible to win.

While designing the game, consider the following:
- Design for specific experience and feelings of the player (awe, tension, zen) to be felt at different points in the game
- Cut superfluous game elements if they do not contribute to the core gameplay loop
- Ensure that all elements are consistent with the input game concept

Come up with the game design with your additions beyond the game concept that is feasible to implement in code using p5.js following the hard constraints. Following are some game elements that you can think of:
- Narrative: the story of the game
- Environment: setting, viewpoint
- Entities: characters: player, NPCs, objects: collectibles, interactive objects, power-ups, destructibles, obstacles etc.
- Mechanics: character abilities, object interactions, rewards, subgoals, final goal
- Graphics: 
  - Visual appearance: the visual appearance of the game environment and entities
  - Animations: the animations of the game entities and their interactions
- progression:
  - Initial phase: teaches mechanics and controls through rewarding, low-challenge gameplay
  - Intermediate phase: Incrementally increase difficulty by introducing more entities, mechanics, and challenges as the player advances
  - Endgame phase: Ensure endgame challenges remain winnable through player skill
  - Transitions between phases must be smooth to allow enough time for the player to adapt to the gradual changes in the game
  - Choose the length of each phase based on the game concept and the game mechanics to make the game engaging and fun for players of different skill levels who play the game for the first time
  - Implement intuitive controls that logically map to appropriate game state changes
  - Introduce entities, objects, and mechanics progressively rather than all at once
  - Design challenges with escalating difficulty that reward players for applying their learned skills and guiding them towards the final goal
  - Gradually introduce complexity, avoiding overwhelming players with difficult situations too early

Include all the elements required by the game concept but feel free to add more elements to make a compelling game. You can add reasonable strategies that the player might use to make progress and win the game.
</process_game_concept_to_game_design>
