- Issue identified: The trajectory line in `Launcher.renderTrajectory` uses `p.beginShape()` and `p.endShape()` without calling `p.noFill()`. Because p5.js retains the fill state from previous drawing operations (like drawing bricks or balls), the trajectory shape is being filled with a color, creating unwanted shaded polygons between the trajectory points.
- Files affected: `entities.js`
- Changes needed: Add `p.noFill()` inside the `renderTrajectory` method of the `Launcher` class before `p.beginShape()` is called. This ensures the trajectory is drawn only as a line (stroke) without any fill.
