- Issue identified: Three separate problems:
  1. W/S movement creates non-normalized vectors when projecting camera direction to horizontal plane, causing weak or inconsistent forward/backward movement
  2. Wall collision detection can push player in wrong direction or apply multiple corrections, causing "teleport on top of walls" effect
  3. Multiple movement keys pressed simultaneously (A+W, S+D, etc.) stack acceleration without normalization, causing excessive speed
  
- Files affected: input.js and physics.js

- Changes needed:
  1. input.js: Properly normalize movement direction after projecting to horizontal plane, and accumulate all movement inputs before applying to avoid stacking acceleration
  2. physics.js: Fix collision resolution to use previous position to determine correct push direction and prevent multiple simultaneous corrections
