- Issue identified: The arrow key events are not calling `event.preventDefault()`, which means the browser's default behavior (page scrolling) is interfering with the game input. When users press UP or DOWN arrows, the browser tries to scroll the page, preventing the game from properly registering these inputs for movement.

- Files affected: input.js

- Changes needed: Add `event.preventDefault()` at the beginning of both `handleKeyDown()` and `handleKeyUp()` functions to prevent the browser from handling arrow keys for scrolling, allowing the game to capture these inputs for player movement.
