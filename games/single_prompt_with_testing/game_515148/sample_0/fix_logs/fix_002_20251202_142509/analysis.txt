- Issue identified: When pressing ENTER on the start screen, the game phase is manually switched to "PLAYING" without calling the initialization function `startNewGame()`. This leaves the game with no player, no level loaded, and empty entity lists, resulting in a blank screen.
- Files affected: `input.js`
- Changes needed: Update the ENTER key handler in `input.js` to set `gameState.shouldReset = true` instead of directly setting the phase. This ensures the main game loop triggers `startNewGame()`, which properly loads the level and creates the player.
