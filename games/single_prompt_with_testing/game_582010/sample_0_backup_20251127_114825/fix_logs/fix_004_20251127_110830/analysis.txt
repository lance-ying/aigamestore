- Issue identified: The movement functions are using both the player's rotation and the camera rotation offset to determine movement direction. This causes incorrect movement behavior. The user wants WASD to move relative to the camera view ONLY, not relative to the player's facing direction. The player should strafe left/right with A/D, and move forward/backward with W/S based solely on where the camera is looking.

- Files affected: entities.js

- Changes needed: Modify the moveForward(), moveBackward(), strafeLeft(), and strafeRight() methods in the Player class to use ONLY gameState.cameraRotationOffset (not gameState.player.rotation + gameState.cameraRotationOffset) when calculating movement direction. This ensures movement is purely relative to the camera's viewing angle, which is controlled by arrow keys.
