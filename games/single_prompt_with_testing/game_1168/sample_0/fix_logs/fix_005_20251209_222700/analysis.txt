- Issue identified: The player falls through the floor because of a flawed optimization in the collision detection logic (`resolvePlatformCollisions` in `physics.js`). The code filters platforms based on the distance between the entity and the platform's origin (top-left corner). Since the main floor platform is very wide (600px), if the player is on the far side, the distance to the origin exceeds the filter threshold (`TILE_SIZE * 3`), causing the floor to be ignored during collision checks.
- Files affected: `physics.js`
- Changes needed: 
  1. Remove the `nearbyPlatforms` filter in `resolvePlatformCollisions` and iterate over all `gameState.platforms`. The number of platforms is small enough that this optimization is unnecessary and harmful.
  2. Corrected the horizontal collision check to use `entity.x` instead of `entity.x + entity.vx`, as `entity.x` has already been updated in the game loop before this function is called.
