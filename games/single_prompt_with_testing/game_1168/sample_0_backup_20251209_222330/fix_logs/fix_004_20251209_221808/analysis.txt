- Issue identified: The player (and certain enemies) fall through platforms due to an incorrect order of operations in the entity's update logic. The `y` position is updated by adding `vy` *before* `resolvePlatformCollisions` is called. However, `resolvePlatformCollisions` then calculates a *predicted* `y` position by adding `vy` *again* to the already moved `y`. This double-counting of `vy` causes the collision check to look too far ahead, potentially skipping over thin platforms entirely in a single frame, leading to the entity falling through.
- Files affected: `entities.js`
- Changes needed: Reorder the Y-axis movement application and platform collision resolution within the `update` method of the `Player` class and the `SLIME` variant of the `Enemy` class. Specifically, `resolvePlatformCollisions` should be called *before* the `this.y += this.vy;` line, allowing the collision resolution to correctly snap the `y` position and zero out `vy` if a collision occurs, or allow `this.y += this.vy;` to apply the movement if no collision is detected.
