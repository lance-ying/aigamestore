- Issue identified: In world.js, when spawning new track and tunnel segments, the code uses `lastSegment.z` (the original spawn position) instead of `lastSegment.mesh.position.z` (the current position after scrolling). As the world scrolls forward, the mesh positions change but the stored `z` property doesn't update. This causes new segments to spawn at incorrect positions, potentially overlapping with existing segments or creating gaps, leading to crashes.

- Files affected: world.js

- Changes needed: Update the spawn logic in `updateWorld()` to use the current mesh position (`segment.mesh.position.z`) instead of the stored z property when calculating where to spawn new segments. This ensures segments spawn at the correct position relative to the scrolling world.
