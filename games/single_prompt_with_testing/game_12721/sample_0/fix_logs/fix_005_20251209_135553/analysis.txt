- Issue identified: The trajectory line was not visible because of a coordinate system mismatch. The `Launcher.render` method translates the drawing origin to the launcher's position, but the `getTrajectoryPreview` function returns global canvas coordinates. When `p.vertex(point.x, point.y)` was called after translation, the trajectory points were effectively offset twice (once by the launcher's position, and then again by the point's global coordinates), causing the line to be drawn far off-screen.
- Files affected: `entities.js`
- Changes needed: Adjust the `p.vertex` calls in `Launcher.renderTrajectory` to account for the `p.translate` applied in `Launcher.render`, by subtracting the launcher's `x` and `y` coordinates from the trajectory points.
