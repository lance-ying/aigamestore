- Issue identified: The player experiences severe slowdown after ~3 seconds because:
  1. Air resistance (0.99 multiplier) is applied to ALL velocity components every frame when not on ground
  2. The player starts at y=1 which is above ground (y=0.8), so they begin in "air" state
  3. After 180 frames (~3 seconds), 0.99^180 ≈ 0.164, reducing velocity to 16%
  4. The friction is per-frame, not frame-rate independent, causing inconsistent behavior

- Files affected: entities.js (Player class update method and constructor)

- Changes needed:
  1. Fix Player constructor to start at proper ground level (y=0.8)
  2. Apply air resistance only to horizontal velocity (x, z), not vertical (y)
  3. Make friction frame-rate independent using Math.pow()
  4. Adjust friction values for better gameplay feel
