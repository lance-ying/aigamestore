- Issue identified: The game freezes (infinite loop) when starting because `level_gen.js` uses a flawed mock object `{random: () => Math.random()}` for random number generation. The `randomInt` utility expects a p5 instance where `random(min, max)` handles range scaling, but the mock ignores arguments and returns 0-1, resulting in `0` after flooring. This causes `len` and `gap` to be 0, trapping the `while` loop in `generateRoom`. Additionally, `particle_system.js` passes `null` to `randomInt`, which would cause a crash when particles are spawned.
- Files affected: `math_utils.js`, `level_gen.js`, `game.js`
- Changes needed: 
    1. Update `math_utils.js` to include a fallback to `Math.random` if the passed `p` object is null or invalid, preventing crashes and fixing logic for non-p5 contexts.
    2. Update `game.js` to pass the valid p5 instance `p` to `generateRoom`.
    3. Update `level_gen.js` to accept `p` in `generateRoom` and use it instead of the broken mock object.
