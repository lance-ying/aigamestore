- Issue identified: In `game.js`, the code attempts to assign properties (`resetGame`, `resetLevel`) to `window.gameInstance` inside the function passed to the `p5` constructor. However, `window.gameInstance` is only assigned the result of `new p5(...)` *after* this function executes. Thus, `window.gameInstance` is `undefined` at the moment of assignment, causing the TypeError.
- Files affected: `game.js`
- Changes needed: Instead of assigning `resetGame` and `resetLevel` to `window.gameInstance` inside the sketch closure, assign them to the p5 instance `p`. The `p` object is the instance that eventually gets assigned to `window.gameInstance`, so the methods will be available globally as intended.
