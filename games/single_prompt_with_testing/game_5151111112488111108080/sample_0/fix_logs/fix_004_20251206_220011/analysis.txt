- Issue identified: The player falls through the initial platform (tunneling) because the collision detection window was fixed at 0.5 units. If the player's fall velocity (which accumulates gravity) caused them to move more than 0.5 units in a single frame, or land exactly in a way that skipped the window, the collision was missed. The high starting position (y=5) exacerbated this by allowing more speed to build up.
- Files affected: `physics.js`, `game.js`
- Changes needed: 
    - In `physics.js`: Implement a dynamic collision threshold based on current vertical velocity (`Math.abs(velocity.y) + 0.1`) to ensure the player is caught even at high speeds.
    - In `game.js`: Lower the player's starting height from 5 to 3 to reduce initial impact velocity and make the start smoother.
