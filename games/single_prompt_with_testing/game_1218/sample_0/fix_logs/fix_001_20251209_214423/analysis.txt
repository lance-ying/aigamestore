- Issue identified: The `physics.js` module is attempting to import `collideRectRect` as a named export from the `p5.collide2d` library URL. The `p5.collide2d` library, when loaded via a `<script>` tag in `index.html`, extends the `p5` prototype, making `collideRectRect` available as a method on `p5` instances (e.g., `p.collideRectRect`). It does not provide ES module named exports, leading to the `Uncaught SyntaxError`.
- Files affected: `physics.js`, `entities.js`
- Changes needed:
    1. In `physics.js`, remove the incorrect ES module import for `collideRectRect`.
    2. In `physics.js`, modify the `checkCollision` and `resolveMapCollisions` function signatures to accept the p5 instance (`p`) as an argument.
    3. In `physics.js`, update the call to `collideRectRect` within `checkCollision` to use `p.collideRectRect`. Also, update the internal call to `checkCollision` within `resolveMapCollisions` to pass the `p` instance.
    4. In `entities.js`, update all call sites for `checkCollision` and `resolveMapCollisions` to pass the `p` instance, which is already available in the `update` methods of `Player`, `Enemy`, and `Projectile`.
