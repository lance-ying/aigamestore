- Issue identified: The `Target.checkCollision` method in `entities.js` calls `s.getWorldCenter(body.pos, body.angle)` on every shape `s` of a physics body. However, `BoxShape` (a subclass of `Shape`) does not implement `getWorldCenter`, causing a crash when a body with a box shape (like 'l' or 'i') interacts with a target.
- Files affected: `physics_core.js`.
- Changes needed: Move the `getWorldCenter` method from `CircleShape` to the base `Shape` class so that all shape types (including `BoxShape`) inherit it. This ensures the method is available for all shapes as required by the gameplay logic.
