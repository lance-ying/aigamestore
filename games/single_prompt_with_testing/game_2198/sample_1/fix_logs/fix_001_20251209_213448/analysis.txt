- Issue identified: The primary reason "only the background shows up" is that p5.js functions like `fill()` and `stroke()` are being passed array arguments (e.g., `[r, g, b]`) directly. While supported in some contexts, this often fails in instance mode or specific versions without spread syntax (e.g., `...[r, g, b]`), resulting in invisible text and entities. Additionally, `particles.js` was rendering particles during the logic update phase before the background was cleared, making them invisible. Finally, `physics.js` contained a crash-inducing bug by calling `window.collideRectPoly` instead of using the p5 instance method, which would crash the game upon level start.
- Files affected: `game.js`, `ui.js`, `entities.js`, `physics.js`, `particles.js`
- Changes needed:
    1.  **`ui.js` & `entities.js`**: Apply spread syntax (`...COLORS.KEY`) to all `p.fill` and `p.stroke` calls to ensure colors are interpreted correctly.
    2.  **`particles.js`**: Split `updateParticles` into separate `updateParticles` (logic) and `renderParticles` (draw) functions to allow correct rendering order.
    3.  **`game.js`**: Reorder the game loop to render particles *after* the background is drawn. Pass the p5 instance `p` to physics functions. Fix background color call.
    4.  **`physics.js`**: Update `checkSpikeCollisions` to accept the p5 instance `p` and use `p.collideRectPoly` instead of the undefined global window function.
