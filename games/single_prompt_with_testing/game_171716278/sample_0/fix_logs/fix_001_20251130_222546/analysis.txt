- Issue identified: When the player collides with the ground, the velocity correction only applies when vDotN > 0 (actively moving into the ground). If the player has been snapped to the surface but still has a downward velocity component, they will penetrate the ground again on the next frame. This causes clipping through the ground and getting stuck.

- Files affected: physics.js

- Changes needed: Remove the conditional `if (vDotN > 0)` check and always project the velocity onto the terrain tangent when the player is on the ground. This ensures the normal (perpendicular) velocity component is always removed, preventing any velocity pointing into the ground from causing repeated penetration.
