- Issue identified: The game loop was using per-frame physics updates without scaling by delta time. On high refresh rate displays (like the 120Hz MacBook Pro), the `gameLoop` runs more frequently, causing gravity, movement, and velocity integration to apply twice as often as intended (relative to a 60Hz standard). This makes the game run too fast.
- Files affected: `game.js`, `physics.js`, `entities.js`.
- Changes needed: 
    1.  In `game.js`, calculate a `timeScale` factor (normalized to 60 FPS) based on `deltaTime`.
    2.  Pass this `timeScale` to all entity update methods and the physics update function.
    3.  In `physics.js`, scale gravity application and velocity integration by `timeScale`.
    4.  In `entities.js`, scale movement speeds, animation smoothing, and timer increments by `timeScale` to ensure consistent speed across different frame rates.
