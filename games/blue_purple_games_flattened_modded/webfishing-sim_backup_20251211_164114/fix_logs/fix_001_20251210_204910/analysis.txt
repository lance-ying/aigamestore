- Issue identified: The `input.update()` method is called at the beginning of the `gameLoop`, before the game logic `update()`. This causes `prevKeys` to be updated to match the current `keys` before the game logic checks for input changes. As a result, `justPressed` (which checks `key && !prevKey`) always returns false because `prevKey` is already equal to `key`.
- Files affected: `game.js` (logic order), `index.html` (CSS styling).
- Changes needed: 
  1. In `game.js`, move `input.update()` to the end of the `gameLoop` function so that input transitions are correctly detected during the frame's update step.
  2. In `index.html`, modify the CSS for `html` and `body` to use Flexbox and 100% width/height to center the game container on the screen.
