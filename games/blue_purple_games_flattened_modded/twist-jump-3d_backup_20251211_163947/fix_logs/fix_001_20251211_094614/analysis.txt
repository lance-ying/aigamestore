- Issue identified: 
    1. Teleporting: The player physics `checkCollisions` method snaps the player to the platform surface if they are below it, regardless of how far below. This causes the player to teleport up to the next platform if they fall into a gap and drift under a platform.
    2. Speed: The game speed accelerates too quickly (`0.0001` per frame) and the base speed might be too high for the user's preference.
    3. Inconsistency: `input.js` hardcodes the reset speed instead of using the global `BASE_SPEED` constant.
- Files affected: `entities.js`, `game.js`, `globals.js`, `input.js`
- Changes needed:
    1. In `entities.js`: Update `checkCollisions` to include a tolerance check (e.g., only snap if within 1.0 unit of the surface).
    2. In `globals.js`: Reduce `BASE_SPEED` from 0.15 to 0.12.
    3. In `game.js`: Reduce speed acceleration from 0.0001 to 0.00002 per frame.
    4. In `input.js`: Import `BASE_SPEED` and use it in `resetGame()` so restarts respect the new speed setting.
