- Issue identified: The `gameState.tree` and `gameState.player` objects are initialized to `null` in `globals.js`. They are only instantiated in the `startGame()` function in `game.js`. However, `startGame()` is not called during the initial page load (only `resetGameState()` is called in `p.setup()`). When the user starts the game by pressing ENTER (handled in `input.js`), the game phase switches to "PLAYING" without ever creating the Tree or Player entities.
- Files affected: `game.js`
- Changes needed: In `game.js`, inside the `p.setup()` function, instantiate `gameState.tree` and `gameState.player` immediately after resetting the game state. This ensures the entities exist for the Start Screen (rendering in the background) and are ready when the user starts the game.
