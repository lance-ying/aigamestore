- Issue identified: 
    1. The level generator was using the player's *current* color to determine safe/wrong obstacles, but failed to account for color changes from upcoming ramps that the player hasn't reached yet. This caused "impossible walls" where the generated safe path matched the old color, resulting in a crash after taking a ramp.
    2. Obstacles were spawning too frequently (every 20 units) for the high game speed (40 units/s), making the game feel too hard.
    3. Obstacles could spawn in the "landing zone" of a ramp jump, causing unfair collisions.
- Files affected: `level_generator.js`
- Changes needed:
    1. Implement `expectedPlayerColor` state in the generator to track predicted color changes.
    2. Update `expectedPlayerColor` when generating a ramp.
    3. Use `expectedPlayerColor` for generating ball walls to ensure they match the future player state.
    4. In `spawnRamp`, update `lastObjectSpawnZ` to reserve space for the jump and trail, preventing overlap.
    5. Increase obstacle spawn spacing from 20 to 30 units.
