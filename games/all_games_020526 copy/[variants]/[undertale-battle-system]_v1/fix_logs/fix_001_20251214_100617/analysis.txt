- Issue identified: The auto-restart feature, triggered after a game over (win or lose), currently returns the player to the initial "press enter to begin" start screen. The user feedback indicates this behavior is undesirable, preferring to jump directly into a new game. The root cause is that the auto-restart `setTimeout` calls `resetGame()`, which explicitly sets the game phase to `GAME_PHASES.START`.
- Files affected: `game.js`
- Changes needed:
  1. Modify the `resetGameState` function to explicitly clear the `autoRestartScheduled` flag and any pending `autoRestartTimeoutId`. This ensures a clean state for these flags regardless of how a game reset/start is initiated.
  2. Change the `setTimeout` callbacks in the `draw` loop for `GAME_PHASES.GAME_OVER_WIN` and `GAME_PHASES.GAME_OVER_LOSE` to call `startGame()` instead of `resetGame()`. This will initiate a new game directly in the `PLAYING` phase, bypassing the `START` screen.
