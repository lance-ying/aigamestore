- Issue identified: The auto-restart functionality, triggered upon game over (win or lose), currently resets the game phase to "START", requiring the player to press ENTER to begin a new game. The user feedback indicates that auto-restart should immediately put the player back into active gameplay (Level 1) without requiring further input.
- Files affected:
    - `game.js`: Contains the auto-restart logic, the `restartGame` function, `loadNextLevel`, `updateHighScore`, and `startGame` (implicitly from `input.js` via `window.loadLevel`).
    - `input.js`: Contains the `startGame` function, `loadNextLevel` function, `updateHighScore` function, and calls to `restartGame` based on user input.
- Changes needed:
    1.  **Centralize Game Start Logic:** Move the core logic for starting a new game (resetting state, loading Level 1, setting `gamePhase` to "PLAYING") into a single exported `startGame` function in `game.js`.
    2.  **Redefine `restartGame`:** Modify the `restartGame` function in `game.js` to simply call this new `startGame` function. This ensures both manual restarts (pressing 'R') and auto-restarts lead directly to a new game at Level 1.
    3.  **Consolidate Level Loading:** Ensure `loadNextLevel` is a single exported function in `game.js`, and `input.js` imports and uses this central version.
    4.  **Consolidate High Score Update:** Ensure `updateHighScore` is a single exported function in `game.js`, and `input.js` imports and uses this central version.
    5.  **Update `input.js`:** Remove redundant `startGame`, `loadNextLevel`, and `updateHighScore` functions. Update `handleKeyPressed` to call the newly centralized and exported functions from `game.js`.
    6.  **Update `window.loadLevel` (dev mode):** Adjust the dev mode `window.loadLevel` in `game.js` to correctly use the new `startGame` and `loadNextLevel` functions.
