- Issue identified: The bird disappears upon launch because it is added to `gameState.activeBirds` but not to `gameState.entities`. The renderer only draws objects present in `gameState.entities`. Furthermore, because the bird is not in `gameState.entities`, the off-screen cleanup logic (in `physics.js`) ignores it. If the invisible bird flies off-screen, it is never marked as inactive, causing the game to wait indefinitely for the bird to "finish", which prevents the next turn or game restart logic from proceeding correctly.
- Files affected: `input.js`
- Changes needed: 
  1. In `launchBird`, push the newly created bird to `gameState.entities`.
  2. In `handleKeyPressed` (for the 'Z' ability), push any new birds (e.g., from the Blue bird split ability) to `gameState.entities`.
