- Issue identified: The automatic game restart after a "Game Over" (win or lose) currently calls `resetToStart`, which incorrectly sends the player back to the initial "press enter to begin" screen (`GAME_PHASES.START`). The user expects the auto-restart to immediately begin a new game in the playing state, similar to pressing ENTER from the start screen.
- Files affected:
    - `game.js`: Needs to change the function called for auto-restart.
    - `controls.js`: Needs refactoring to provide distinct functions for resetting to the start screen and restarting directly into gameplay, and to ensure a full state reset for starting a new game.
- Changes needed:
    - In `controls.js`, create a private helper function `_fullResetAndSetPhase` to centralize all game state resetting logic (score, lives, entities, player, timers, etc.) and set the `gamePhase`.
    - Modify `startGame` to use `_fullResetAndSetPhase` and set the phase to `PLAYING`.
    - Create a new `export` function `resetToStartScreen` (replacing the old `resetToStart`) that uses `_fullResetAndSetPhase` and sets the phase to `START`. This will be used for the manual 'R' key press.
    - Create a new `export` function `restartGamePlaying` that uses `_fullResetAndSetPhase` and sets the phase to `PLAYING`. This will be used for the automatic restart.
    - In `handleKeyPressed`, update the 'R' key logic to call `resetToStartScreen`.
    - In `game.js`, update the import statement to include `restartGamePlaying` and change the auto-restart logic to call `restartGamePlaying`.
