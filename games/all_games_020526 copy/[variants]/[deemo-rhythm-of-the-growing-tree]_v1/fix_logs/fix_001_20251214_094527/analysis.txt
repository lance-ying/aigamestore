- Issue identified: The automatic restart after a game over (win/lose) currently calls `gameLogic.restartGame()`, which transitions the game to the `PHASE_START` screen. The user expects the auto-restart to immediately begin a new game from Level 1, bypassing the start screen.
- Files affected:
    - `game.js`: The `setTimeout` callback for auto-restart needs to call a different function.
    - `game_logic.js`: A new function is needed to handle the "reset all progress and start a new game" behavior for auto-restart, distinct from the existing `restartGame()` which goes to the start screen.
- Changes needed:
    - In `game.js`, modify the `setTimeout` in the `PHASE_GAME_OVER_WIN` and `PHASE_GAME_OVER_LOSE` blocks to call a new `gameLogic.autoRestartGame()` method.
    - In `game_logic.js`, create `autoRestartGame()` which will reset all global game statistics (score, combo, tree height, story progress, unlocked songs) and then call the existing `startGame()` method to load the first level and transition to `PHASE_PLAYING`. The existing `restartGame()` will remain unchanged for manual 'R' key presses to return to the start screen.
