- Issue identified:
    1.  `Uncaught ReferenceError: drawStartScreen is not defined` in `game.js`. This occurs because the `drawStartScreen` function, which is defined and exported from `rendering.js`, is not imported into `game.js` where it is being called. The same applies to `drawGame` and `drawGameOver` which are used but not imported.
    2.  Automatic game restart after "Game Over" or "Victory" incorrectly returns the player to the "Start" screen instead of immediately beginning a new game. The current auto-restart logic calls `resetGame`, which is designed to return to the title screen.

- Files affected:
    - `game.js`
    - `input.js`
    - `globals.js` (minor update to remove `autoRestartTimer` from reset logic in `globals.js` if it's moved to `input.js`, but it's fine as a state variable, just the setting of it needs to be managed)

- Changes needed:
    1.  **`game.js`**:
        *   Import `drawStartScreen`, `drawGame`, and `drawGameOver` from `rendering.js`.
        *   Import the new `autoRestartGame` function from `input.js`.
        *   Change the auto-restart call from `resetGame(p)` to `autoRestartGame(p)`.
    2.  **`input.js`**:
        *   Refactor `resetGame` into a private helper `_softResetGameState(p)` that handles all state variable resets *except* setting the `gamePhase` and `currentLevel`. This function will also clear `gameState.autoRestartTimer`.
        *   Modify the existing `resetGame(p)` to call `_softResetGameState(p)`, then explicitly set `gameState.gamePhase` to `GAME_PHASES.START` and `gameState.currentLevel` to `0`.
        *   Create a new `export function autoRestartGame(p)` that calls `_softResetGameState(p)` and then `startGame(p)` to immediately begin a new game.
        *   Export `startGame(p)` so it can be called by `autoRestartGame`.
    3.  **`globals.js`**: (No changes needed here. `autoRestartTimer` is a state variable and its reset logic will be handled in `input.js`.)
