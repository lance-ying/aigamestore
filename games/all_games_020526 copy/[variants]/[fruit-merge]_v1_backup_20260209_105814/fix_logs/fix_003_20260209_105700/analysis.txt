- Issue identified: The pause/unpause functionality, triggered by the 'ESC' key, is glitchy and spams between states when the key is held down. This indicates that the game is processing repeated `keydown` events for the 'ESC' key, instead of only the initial press.
- Files affected: `input.js`
- Changes needed: Modify the `handleKeyDown` function in `input.js` to ensure that toggle-like actions (like pause/unpause, start game, restart game) only trigger on the initial `keydown` event, ignoring subsequent `keydown` events generated by holding the key down (key repeat). This will be achieved by using the internal `keys` state to track if a toggle key is already logically "down". The existing `e.repeat` check will be removed as this new approach is more robust for toggle actions.
