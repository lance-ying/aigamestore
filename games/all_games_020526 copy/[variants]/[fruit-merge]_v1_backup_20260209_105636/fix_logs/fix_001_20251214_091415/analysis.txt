- Issue identified: The automatic game restart after a "Game Over" event incorrectly returns the player to the "START" screen (requiring them to press ENTER again) instead of directly restarting gameplay. This is because the `endGame` function's auto-restart timer calls `restartGame()`, which explicitly sets `gameState.gamePhase` to 'START'.
- Files affected:
    - `game.js`: The `startGame()` and `restartGame()` functions need a common helper to clear auto-restart flags, and `restartGame()`'s logic for clearing these flags needs to be moved.
    - `physics.js`: The `endGame()` function needs to call `startGame()` instead of `restartGame()` for the auto-restart scenario.
- Changes needed:
    1.  In `game.js`, create a new internal helper function `_clearAutoRestartFlags()` to encapsulate the logic for clearing the `autoRestartTimeoutId` and `autoRestartScheduled` flag.
    2.  Call `_clearAutoRestartFlags()` at the beginning of both `startGame()` and `restartGame()` to ensure any new game or manual restart properly cancels a pending auto-restart.
    3.  In `physics.js`, modify the `endGame()` function's auto-restart `setTimeout` callback to call `startGame()` instead of `restartGame()`. This leverages `startGame()`'s existing logic to reset the game state and immediately transition to the 'PLAYING' phase.
