- Issue identified: The pause/unpause functionality is glitchy because the `togglePause` function is called repeatedly as long as the ESC key is held down. This is due to the pause logic being placed within `handleInput`, which runs every frame and checks the continuous state of the key using `isKeyPressed`.
- Files affected: input.js
- Changes needed: Move the `togglePause` call from the per-frame `handleInput` function to the `registerKeyPress` function, which is triggered only once when a key is initially pressed. This will ensure the pause state is toggled only once per key press.
