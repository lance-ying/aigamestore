- Issue identified: The game lacks a dedicated "Game Over" screen for both win and lose conditions. When the player dies or wins, the game currently attempts an "auto-restart" that transitions directly back to the "Start" screen after a delay, which the user finds undesirable. The user wants to see a Game Over screen and then manually initiate a restart.
- Files affected:
    - `globals.js`: To remove unused auto-restart state variables.
    - `rendering.js`: To implement and display the "Game Over" screen for win/lose phases.
    - `game_logic.js`: To remove the call to schedule auto-restart when the player dies.
    - `input.js`: To remove auto-restart scheduling/cancellation logic, and to add 'R' key handling for manual restart from the new "Game Over" screen.
- Changes needed:
    - Create a `renderGameOver` function in `rendering.js` to display the game over state, final score, high scores, and restart instructions.
    - Update `renderGame` in `rendering.js` to call `renderGameOver` for `GAME_PHASES.GAME_OVER_WIN` and `GAME_PHASES.GAME_OVER_LOSE`.
    - Remove `autoRestartTimeoutId` and `autoRestartScheduled` from `gameState` in `globals.js`.
    - Remove the `scheduleAutoRestart` and `cancelAutoRestart` functions from `input.js`.
    - Remove all calls to `scheduleAutoRestart` and `cancelAutoRestart` from `game_logic.js` and `input.js`.
    - Add 'R' key handling in `input.js` for `GAME_PHASES.GAME_OVER_WIN` and `GAME_PHASES.GAME_OVER_LOSE` to call `resetToStart`.
