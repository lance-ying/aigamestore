- Issue identified: The game's pause/unpause functionality, triggered by the ESC key, is glitchy and "spams" between states when the key is held down. This occurs because the existing logic to prevent repeat key actions (`keys[`action_${keyCode}`]`) is not robust enough or is being bypassed, allowing multiple `keydown` events (generated by the browser when a key is held) to toggle the pause state.
- Files affected: `input.js`
- Changes needed: Implement a more reliable check for key repetition using the `e.repeat` property of the KeyboardEvent object within the `handleKeyDown` function. This will ensure that action keys like ESC only trigger their associated function once per physical key press, ignoring subsequent `keydown` events while the key is held. The redundant custom `action_` key state tracking can be removed.
