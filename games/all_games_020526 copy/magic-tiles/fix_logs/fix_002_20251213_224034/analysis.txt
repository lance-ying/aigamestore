- Issue identified: The game's auto-restart functionality, triggered after a game over (win or lose), incorrectly transitions the game back to the "START" screen, requiring the user to press ENTER again. The desired behavior is to immediately restart the game into the "PLAYING" state. This is because the auto-restart was calling `resetGame()`, which explicitly sets the game phase to "START".
- Files affected: `game.js`
- Changes needed: Modify the `setTimeout` callbacks for auto-restart in the `GAME_OVER_WIN` and `GAME_OVER_LOSE` phases to call `startGame()` instead of `resetGame()`. The `startGame()` function already performs all necessary game state initialization and sets the phase to "PLAYING", which is the desired behavior for an automatic restart.
