- Issue identified: The `maxInstances` limit in the `VoxelWorld` class is set to 5000. In a 64x64 world with terrain, the number of visible blocks (especially Dirt and Stone) significantly exceeds this limit. As a result, many blocks (including newly placed Stone or Brick blocks if the buffer is full) are not rendered, appearing invisible and causing "glitches" (holes in the terrain).
- Files affected: `voxel_world.js`
- Changes needed: Increase `maxInstances` from 5000 to 50000 to ensure all visible blocks in the chunk can be rendered.
