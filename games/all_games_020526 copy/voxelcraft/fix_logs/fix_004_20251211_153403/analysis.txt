- Issue identified: The `InstancedMesh` used for rendering blocks has its bounding volume calculated based on the base geometry (a 1x1x1 box at the origin) by default. When the player looks away from the origin, Three.js frustum culling determines the mesh is off-screen and stops rendering it, causing all instances (blocks) of that type to disappear.
- Files affected: `voxel_world.js`
- Changes needed: In `initMeshes`, set `mesh.frustumCulled = false` for each block type's `InstancedMesh`. This prevents Three.js from culling the mesh based on the incorrect bounding volume, ensuring blocks remain visible regardless of camera angle.
