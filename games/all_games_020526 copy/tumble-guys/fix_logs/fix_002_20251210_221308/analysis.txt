- Issue identified: 
    1. Jumping is unreliable because the `onGround` flag flickers. The collision detection logic requires exact contact or penetration (`dist < radius`) which fails when gravity isn't applied on grounded frames (preventing penetration), causing the player to float slightly and lose grounded status in alternating frames.
    2. Diving gets stuck because the player slides along the ground without triggering the `onGround` state due to the same strict collision threshold. The state machine relies on `onGround` to transition from 'DIVE' to 'STUMBLE'.
    3. There is a bug in `physics.js` where vector mutation (`multiplyScalar`) corrupts the collision normal, leading to incorrect velocity resolution.
- Files affected: `physics.js`
- Changes needed:

    2. Fix vector math to use `.clone()` before mutating vectors to prevent calculation errors.
