2. "but I also don't want the hud/cursor/controls text to disappear when the game is paused"

Looking at the code:

In `game.js`, the draw function has:
```javascript
} else if (gameState.gamePhase === PHASE_PAUSED) {
    renderGame(p);
    // renderPaused(p) removed to eliminate visual overlay and text
}
```

So it seems like `renderPaused(p)` was already removed (there's a comment about it).

But let me check `ui.js` to see if there's a renderPaused function defined... No, there isn't one in the provided files.

So currently when paused:
- The game is rendered (renderGame)
- No overlay is shown
- But the UI (renderUI) is NOT called when paused

The issue is that when the game is paused, `renderUI(p)` is not called. Looking at the draw function:

```javascript
if (gameState.gamePhase === PHASE_START) {
    renderStartScreen(p);
} else if (gameState.gamePhase === PHASE_PLAYING) {
    updateGame(p);
    renderGame(p);
    renderUI(p);
} else if (gameState.gamePhase === PHASE_PAUSED) {
    renderGame(p);
    // renderPaused(p) removed to eliminate visual overlay and text
} else if (gameState.gamePhase === PHASE_GAME_OVER_LOSE || gameState.gamePhase === PHASE_GAME_OVER_WIN) {
    renderGame(p);
    renderGameOver(p);
}
```

When paused, only `renderGame(p)` is called, but NOT `renderUI(p)`. This means the HUD (score, coins) disappears when paused.


The fix is straightforward: I need to call `renderUI(p)` during the paused phase so the score and coins remain visible. The controls text lives in HTML so it's always there, and the cursor is browser-managed, so both of those will persist automatically.
