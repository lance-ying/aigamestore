- Issue identified: The game loop was using a variable `deltaTime` to drive physics updates, but the force applications (gravity, input) were constant per update. On high refresh rate screens (like the user's MacBook Pro at 120Hz), `updatePhysics` was called more often per second, causing gravity and input forces to be applied more frequently, making the game run faster and physics feel heavier/jittery. Additionally, the jump mechanic was not "floaty" enough.
- Files affected: `game.js`, `globals.js`
- Changes needed: 
  1. In `game.js`, implemented a fixed timestep accumulator (1/60s). Logic and physics now run at a consistent 60Hz regardless of the display refresh rate. This prevents the "speed up" on high-refresh screens.
  2. In `globals.js`, reduced `GRAVITY` from `-0.015` to `-0.011`. This reduces the downward force, making the jump arc longer and slower ("floatier") as requested.
