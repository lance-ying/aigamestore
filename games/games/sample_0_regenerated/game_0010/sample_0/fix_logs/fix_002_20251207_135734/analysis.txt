- Issue identified: The `keydown` and `keyup` event listeners in `inputs.js` were inconsistently processing the `e.key` value. Specifically, for keys with `e.key.length > 1` (e.g., "Enter", "Shift", "ArrowUp"), the `e.key.toLowerCase()` method was not being applied, leaving them in their original case (e.g., "Enter"). However, the comparison logic (`if (key === 'enter')`) was expecting a lowercase string. This mismatch caused the game to not register inputs from these keys, including the 'Enter' key to start the game.
- Files affected: `inputs.js`
- Changes needed: Update the input processing logic in `inputs.js` to consistently convert all `e.key` values to lowercase. Subsequently, update the comparison strings for affected keys (specifically the arrow keys) to match their lowercase versions.
