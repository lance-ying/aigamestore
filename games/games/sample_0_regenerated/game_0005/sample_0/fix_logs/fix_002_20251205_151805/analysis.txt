- Issue identified: The `p5.collide2d` library is being imported as an ES module using a URL that results in a `404` error (`net::ERR_ABORTED 404`). This is because `p5.collide2d` is typically distributed as a global script that extends the `p5.prototype`, not as an ES module with named exports. The `?module` query parameter on the CDN URL is likely causing the server to look for a module-specific file or header, which isn't present, leading to the error.
- Files affected: `index.html`, `entities.js`, `utils.js`.
- Changes needed: Revert to loading `p5.collide2d` as a global script in `index.html`. Remove the ES module import statements from `entities.js` and `utils.js`. Update calls to `collideRectRect` and `checkAABB` in `entities.js` and `utils.js` to access the collision function via the `p5` instance (`p.collideRectRect`) as intended by the library's design. The `checkAABB` utility function will also need to accept the `p5` instance as an argument.
