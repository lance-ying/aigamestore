- Issue identified: The physics `moveX` and `moveY` functions in `physics.js` contain a logical error in how they apply movement after collision detection. When a collision with a solid is detected, the entity's position is corrected (snapped to the solid's edge), and its velocity is set to zero. However, the original `amount` of movement for the current frame is then *still added* to the entity's position after the collision check loop, causing the entity to pass through solids. This results in the player falling through the initial floor and dying.
- Files affected: `physics.js`
- Changes needed: Refactor `moveX` and `moveY` to calculate a `finalX` and `finalY` position respectively. Initialize these with the intended movement (`entity.x + amount` or `entity.y + amount`). If a collision is detected with any solid, adjust this `finalX` or `finalY` to the exact collision point. After checking all solids, apply the calculated `finalX` or `finalY` to the entity's actual position. This ensures that movement is correctly clamped by collisions.
