- Issue identified: The p5.js canvas does not automatically receive keyboard focus when the game loads. This prevents `p.keyIsDown()` from registering key presses, such as the Enter key, on the initial "START" screen. While the `setMode` function in `index.html` attempts to focus the canvas, this function is only triggered by clicking UI buttons, not by the initial game load or by pressing Enter.
- Files affected: `game.js`
- Changes needed: Add a call to `p.canvas.focus()` within the `p.setup` function to ensure the canvas has keyboard focus from the moment it's initialized, allowing input to be registered immediately.
