- Issue identified: The `handleInput` function in `input.js` returns the `keys` object directly, but `game.js` and `entities.js` expect it to return a wrapper object `{ keys: ... }`. This causes `input.keys` to be undefined, throwing the TypeError when accessing `input.keys.jumpPressed`.
- Files affected: `input.js`
- Changes needed: 
  1. Modify `handleInput` to return `{ keys }` instead of `keys` to match the expected data structure in `game.js`.
  2. Introduce an `inputBuffer` to properly capture `jump` and `interact` key presses from the `handleKeyPressed` event, ensuring they persist until the `handleInput` loop processes them (fixing a logic bug where inputs were cleared before being read).
