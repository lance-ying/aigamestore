- Issue identified: The input handling logic for "one-shot" actions (Jump, Attack, Dash) was flawed. The `updateInput` function, called at the start of every frame, was clearing the `keyPressedThisFrame` object. Since browser input events (like `keyPressed`) occur asynchronously between frames, any key presses recorded were immediately wiped before the game logic (`Player.update`) could read them. This prevented `isKeyPressed` from returning true for human inputs.
- Files affected: `input.js`
- Changes needed: Introduced a `keyEventsBuffer` to capture key presses as they happen. Modified `updateInput` to transfer these buffered events into `keyPressedThisFrame` at the start of the frame, ensuring they persist for exactly one logic update cycle. Also updated the automated testing logic to correctly set `keyPressedThisFrame` so bots can also jump and attack.
