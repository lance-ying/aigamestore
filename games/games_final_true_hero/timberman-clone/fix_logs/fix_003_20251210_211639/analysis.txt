- Issue identified: The random tree generation logic allows for "impossible" patterns, specifically a branch on one side immediately followed by a branch on the opposite side (e.g., Segment 0 has a Right Branch, Segment 1 has a Left Branch). In this scenario, the player must be on the Left to chop Segment 0 safely. However, as soon as they chop, Segment 1 (Left Branch) falls instantly. Since the player is on the Left, they are immediately squashed. They cannot move to the Right before chopping because Segment 0 (Right Branch) blocks them. This creates an unavoidable death which players perceive as dying "incorrectly" despite playing properly.
- Files affected: `entities.js`
- Changes needed: Update the `addNewRandomSegment` method in the `Tree` class to check the previous segment. If the previous segment has a branch, the new segment must not have a branch on the opposite side. It should either be empty or on the same side (stacked branches are playable).
