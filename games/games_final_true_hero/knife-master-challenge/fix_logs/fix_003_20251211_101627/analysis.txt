- Issue identified: 
    1. `resetGame()` did not clear `gameState.target`. This prevented `setupLevel()` from running when restarting, leaving the game with an old target, 0 remaining knives, and no active knife to throw. This also meant no new apples were spawned.
    2. The rendering logic for stuck knives in `Target.render` used incorrect coordinate transformations (`translate(0, radius)` instead of `translate(radius, 0)` after rotation), causing knives to appear at the wrong position (offset by 90 degrees/on the side) relative to where they hit.
- Files affected: `globals.js`, `entities.js`
- Changes needed:
    1. In `globals.js`: Update `resetGame()` to set `gameState.target = null`.
    2. In `entities.js`: Update `Target.render()` to correctly transform coordinates for stuck knives using standard polar coordinates (`rotate(angle)`, `translate(radius, 0)`) and adjust the knife rotation to point outward correctly.
