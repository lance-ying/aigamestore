- Issue identified: The `p.keysPrev` (previous key state) is updated to match `p.keysDown` (current key state) *before* input handling occurs in the `draw` loop. This causes `!p.keyWasDown(code)` to always return false when a key is held, preventing the "just pressed" logic (rising edge detection) from triggering. Consequently, the Enter key press is never detected as a new press, and the game start condition fails.
- Files affected: `game.js`
- Changes needed: 
  1. Move the `p.keysPrev` update loop from the beginning of `p.draw` to the very end of `p.draw`. This ensures that during `handleInput`, `p.keysPrev` represents the state of the *previous* frame, allowing `!p.keyWasDown` to correctly return true for a newly pressed key.
  2. Initialize `gameState.lastFrameTime` in `p.setup` to prevent a large `deltaTime` on the first frame.
