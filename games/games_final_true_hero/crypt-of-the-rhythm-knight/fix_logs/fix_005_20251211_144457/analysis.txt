- Issue identified:
    1.  **Unfair Damage**: The player takes damage immediately upon missing a beat (stumbling), which the user dislikes.
    2.  **Damage Logic**: The user wants enemies to *only* be able to damage the player if the player missed the beat. If the player is on-beat, they should be safe (block/dodge).
    3.  **UI Overlap**: The heart display overlaps with the score display because the hearts take up too much width.
    4.  **UI Sync**: The heart display (UI) does not update because it reads from `gameState.health`, but damage was only applying to `gameState.player.health`.
- Files affected: `globals.js`, `input.js`, `physics.js`, `ui.js`
- Changes needed:
    1.  `globals.js`: Add `playerMissedBeat` flag to `gameState`.
    2.  `input.js`: Remove `physics.damagePlayer` on "MISS". Instead, set `playerMissedBeat = true`. Set it to `false` on success.
    3.  `physics.js`:
        -   In `handleEnemyCollision`, check `playerMissedBeat`. If false, block attack (no damage). If true, apply damage.
        -   In `damagePlayer` and `collectItem`, sync `gameState.health` with `gameState.player.health` so the UI updates correctly.
    4.  `ui.js`: Shift Score, Multiplier, and Combo text to the right to accommodate the width of the heart display.
