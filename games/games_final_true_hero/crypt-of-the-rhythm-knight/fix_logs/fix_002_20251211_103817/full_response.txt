<analysis>
- Issue identified: When the player dies and restarts the game, the `gameState.player` object is not cleared. The `startLevel` function checks for an existing `gameState.player` to preserve health/score between levels. Because the old player object (with <= 0 health) persists, the new game starts with a player having 0 health. Thus, the first hit from any enemy (like the white Skeleton) kills the player instantly.
- Files affected: `globals.js`
- Changes needed: Update `resetGameState` in `globals.js` to explicitly set `gameState.player = null`. This ensures that `startLevel` initializes a fresh player with full health on a new game, while still allowing `nextLevel` (which doesn't call reset) to preserve stats.
</analysis>
