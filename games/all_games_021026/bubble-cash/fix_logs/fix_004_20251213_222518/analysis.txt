- Issue identified: The automatic game restart after a 'GAME_OVER_WIN' or 'GAME_OVER_LOSE' state incorrectly transitions the game back to the 'START' screen, requiring the player to press 'Enter' again. The user feedback indicates this auto-restart should bypass the start screen and immediately begin a new game. The manual restart triggered by the 'R' key should still go to the 'START' screen.
- Files affected:
  - `game.js`
  - `input.js`
- Changes needed:
  - Modify the `restartGame` function in `input.js` to accept a `targetPhase` parameter. This parameter will determine whether the game restarts to the 'START' screen (default, for manual 'R' key restart) or directly to the 'PLAYING' phase (for auto-restart).
  - Update the auto-restart `setTimeout` calls in `game.js` to pass 'PLAYING' as the `targetPhase`.
  - Update the manual 'R' key handler in `input.js` to explicitly pass 'START' as the `targetPhase`.
  - Add logic within `restartGame` to reset `gameState.canFire`, `gameState.swapAvailable`, and the player's `shooterAngle` for a clean state, regardless of the target phase.
  - Update `window.loadLevel` in `game.js` to restart to 'PLAYING' phase.
