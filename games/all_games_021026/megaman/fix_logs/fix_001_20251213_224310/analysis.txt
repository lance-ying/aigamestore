- Issue identified: The P5.js game instance is being initialized at the top level of a `type="module"` script (`game.js`). While `type="module"` scripts are deferred, there can still be subtle timing issues with P5.js's global `p5` object availability or the readiness of the DOM (specifically `document.body`) when the `new p5()` constructor is called. This often results in the canvas not being created or rendered, leading to a blank screen.
- Files affected: `game.js`
- Changes needed: Wrap the P5.js instance creation (`new p5(...)`) and its subsequent global exposures (`window.gameInstance`, `window.loadLevel`) within a `window.onload` event listener in `game.js`. This ensures the P5.js sketch is initialized only after the entire page (including the P5.js library and the DOM) has fully loaded. The `gameInstance` variable will be declared at the module scope to allow assignment within the `onload` handler.
