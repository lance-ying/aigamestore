- Issue identified: The root cause of the flickering pause state is that the pause toggle logic was implemented in `handleInput`, which is executed every frame within the `draw` loop. It checked if the Esc key was currently held down using `isKeyPressed`. Consequently, as long as the key was held, the game state would toggle between "PLAYING" and "PAUSED" every single frame (60 times per second).
- Files affected: `input.js`
- Changes needed: Remove the redundant phase transition logic (Enter for Start, Esc for Pause, and R for Restart) from the `handleInput` function. These actions are already correctly handled in the `registerKeyPress` function, which is triggered by the p5.js `keyPressed` event. This event fires once per physical key press (or at the OS-level repeat rate), preventing the frame-by-frame toggling. Added logging to the `togglePause` helper function to maintain consistency with the removed code.
