The user wants to make the game freeze when paused, but without:
1. Showing any text (no "PAUSED" message)
2. Graying out the screen
3. Hiding the HUD (the HUD should stay visible)

Currently, when the game is paused (gameState.gamePhase === "PAUSED"), the ui.js file has a case for "PAUSED" that currently does nothing (just a comment saying we skip rendering to keep the screen clean).

Looking at the code:
1. In game.js, when gamePhase is "PAUSED", the updatePlaying() function is not called, so the game is already frozen
2. In ui.js, the renderUI() function has a case for "PAUSED" that currently does nothing
3. The HUD is only rendered when gamePhase is "PLAYING"

So the issue is:
- The game already freezes (updatePlaying doesn't run)
- But the HUD disappears because renderHUD() is only called in the "PLAYING" case
- The user wants the HUD to stay visible while paused

The fix is simple: In ui.js, when the game is paused, we should render the HUD but not show any overlay or "PAUSED" text.

Let me check the renderUI function:
```javascript
export function renderUI() {
    if (!uiContext) return;
    
    // Clear
    uiContext.clearRect(0, 0, CANVAS_WIDTH, CANVAS_HEIGHT);
    
    // Render based on Phase
    switch (gameState.gamePhase) {
        case "START":
            renderStartScreen();
            break;
        case "PLAYING":
            renderHUD();
            break;
        case "PAUSED":
            // Show NO visual overlay, NO "PAUSED" text, NO instructions
            // We skip rendering anything here to keep the screen clean while frozen
            renderHUD();
            break;
        case "GAME_OVER_WIN":
        case "GAME_OVER_LOSE":
            renderHUD(); // Show final score behind
            renderGameOverScreen();
            break;
    }
}
```

The solution is straightforward—just call `renderHUD()` in the "PAUSED" case to keep the HUD visible while the game state remains frozen.
