2. In `ui.js`, the `drawUI(p)` function calls `drawPausedOverlay(p)` when the game phase is 'PAUSED':

3. In `gameLogic.js`, the `updateGame(p)` function only runs when `gameState.gamePhase !== 'PLAYING'`, so when paused, the game logic doesn't update, which is correct.

4. In `render.js`, the `renderGame(p)` function handles different game phases. When PAUSED, it should just continue rendering the game as normal without the overlay.
