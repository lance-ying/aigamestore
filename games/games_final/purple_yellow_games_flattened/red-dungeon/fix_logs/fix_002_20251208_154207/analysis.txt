- Issue identified: The user wants the score to reflect how far 'up' the player has progressed from their starting position, rather than just using the absolute negative Y grid coordinate. The current implementation effectively starts counting score from `gy=0` (world Y=0), which might not align with progress from the actual spawn point.
- Files affected:
    - `globals.js`: To add a new state variable `initialPlayerGy` and reset it.
    - `level_gen.js`: To set the `initialPlayerGy` when the player is first spawned.
    - `entities.js`: To modify the player's score calculation to use `initialPlayerGy`.
- Changes needed:
    - Add `initialPlayerGy` to the `gameState` object in `globals.js`.
    - Initialize `initialPlayerGy` in the `resetGame` function in `globals.js`.
    - In `level_gen.js`, after the player is created in `setupLevel`, assign the player's starting `gy` to `gameState.initialPlayerGy`.
    - In `entities.js`, update the `Player.update` method to calculate the score as the difference between `initialPlayerGy` and the player's current `gy`.
